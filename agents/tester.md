# tester: 테스트 에이전트

---
model: sonnet
type: subagent
triggers: [테스트, test, 검증, 단위테스트, 통합테스트]
---

테스트 작성 및 실행을 담당하는 서브에이전트입니다.

## 메타데이터

| 항목 | 값 |
|------|-----|
| **모델** | `sonnet` (테스트 작성에 적합) |
| **컨텍스트** | 별도 (테스트 대상 코드만 전달) |
| **용도** | 테스트 케이스 도출 및 작성 |

## 사용 시점

- 구현 완료 후 테스트 작성
- 기존 테스트 보완
- 버그 재현 테스트
- TDD 사이클

## 호출 방법

```
Task tool 사용:
- subagent_type: "general-purpose"
- model: "sonnet"
- prompt: 테스트 대상 + 요구사항
```

## wb-check 통합

`/wb-check` 실행 시 code 유형 서브태스크는 자동으로 tester를 호출합니다:

```
/wb-check
├─ [1] checker (opus) → Match Rate 계산
└─ [2] tester (sonnet) → 테스트 커버리지 분석
    ├─ 신규 코드 테스트 검토
    ├─ 커버리지 계산
    └─ 권장 테스트 제시
```

### 결과 통합

| Match Rate | Coverage | 판정 |
|-----------|----------|------|
| >= 90% | >= 80% | ✅ Green light |
| >= 90% | < 80% | ⚠️ 테스트 부족 |
| 70-89% | any | ⚠️ Gap 수정 필요 |
| < 70% | any | ❌ 재작업 |

## 테스트 유형

### 단위 테스트
```
"다음 함수의 단위 테스트를 작성해주세요:
- 함수: {함수 코드}
- 케이스: 정상, 에러, 엣지"
```

### 통합 테스트
```
"다음 API의 통합 테스트를 작성해주세요:
- 엔드포인트: {API 정보}
- 시나리오: {테스트 시나리오}"
```

### E2E 테스트
```
"다음 플로우의 E2E 테스트를 작성해주세요:
- 플로우: {사용자 플로우}
- 도구: {Playwright/Cypress 등}"
```

```markdown
{참고할 기존 테스트 패턴}
```

## 반환 형식

```markdown
## 테스트 결과

### 작성된 테스트
- {테스트 파일}: {케이스 수}개

### 커버리지
- 정상 케이스: ✅
- 에러 케이스: ✅

- 실패: Y개

### 실패 분석
{실패 원인 및 수정 제안}
```
