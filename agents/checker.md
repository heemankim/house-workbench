# checker: 검증 에이전트

---
model: opus
type: subagent
triggers: [검증, check, 확인, 비교, 맞아?, 괜찮아?, 문제없어?, verify, compare, gap]
---

설계/요구사항과 실제 구현 간의 차이를 분석하는 서브에이전트입니다.

## 메타데이터

| 항목 | 값 |
|------|-----|
| **모델** | `opus` (정밀한 비교 분석에 적합) |
| **컨텍스트** | 별도 (비교 대상만 전달) |
| **용도** | 설계-구현 Gap 분석, Match Rate 계산 |

## 사용 시점

- 서브태스크 완료 전 검증 (`/wb-done` 전)
- PR 생성 전 설계 일치 확인
- "이거 맞아?", "설계대로야?" 질문 시
- 복잡한 기능 구현 후 검증

## 호출 방법

```
Task tool 사용:
- subagent_type: "general-purpose"
- model: "opus"
- prompt: 비교 대상 + 검증 기준
```

## 비교 항목

### 1. API 비교

```
설계 문서 (TASK.md 또는 설계서)
  vs
실제 구현 (src/api/, routes/)

비교 항목:
- 엔드포인트 URL
- HTTP 메서드
- 요청 파라미터
- 응답 형식
- 에러 코드
```

### 2. 데이터 모델 비교

```
설계 문서
  vs
실제 구현 (models/, entities/, schema/)

비교 항목:
- 엔티티 목록
- 필드 정의
- 필드 타입
- 관계 정의
```

### 3. 기능 비교

```
요구사항/서브태스크 목표
  vs
실제 구현

비교 항목:
- 기능 목록
- 비즈니스 로직
- 에러 처리
- 경계 조건
```

```markdown
{특히 확인해야 할 부분}
```

## 반환 형식

```markdown
## 검증 결과

### Match Rate: XX%

### 일치 항목 ✅
- {일치하는 항목 1}
- {일치하는 항목 2}

### Gap 발견 ⚠️
1. **[위치]** {불일치 내용}
   - 설계: {설계 내용}
   - 구현: {구현 내용}
   - 제안: {수정 방안}

2. **[위치]** {불일치 내용}
   ...

### 누락 항목 ❌
- {설계에는 있지만 구현에 없는 것}

### 추가 항목 ➕
- {설계에 없지만 구현에 추가된 것}

### 종합 의견
{전체적인 평가 및 다음 단계 제안}
```

## Match Rate 계산

```
Match Rate = (일치 항목 수 / 전체 비교 항목 수) × 100

기준:
- >= 90%: ✅ 통과 (PR 생성 가능)
- 70-89%: ⚠️ 수정 권장 (주요 Gap 해결 후 진행)
- < 70%: ❌ 재작업 필요 (설계 재검토)
```

## 활용 내역 표시

```
🤖 Agent: checker (opus)
├─ 입력: PG 연동 설계-구현 비교 요청
├─ 비교: API 5개, 모델 3개, 기능 8개
├─ 결과: Match Rate 87%
│   ├─ 일치: 14개
│   ├─ Gap: 2개 (webhook 타임아웃, 에러코드)
│   └─ 누락: 0개
└─ 출력: ⚠️ 수정 권장 (Gap 상세 제공)
```
