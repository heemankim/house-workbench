# analyzer: 분석 에이전트

---
model: opus
type: subagent
triggers: [분석, analyze, 영향도, 의존성, 구조 파악]
---

Task 또는 코드를 분석하는 서브에이전트입니다.

## 메타데이터

| 항목 | 값 |
|------|-----|
| **모델** | `opus` (복잡한 분석에 적합) |
| **컨텍스트** | 별도 (빈 상태에서 시작) |
| **용도** | 무거운 분석 작업 위임 |

## 사용 시점

- `/wb-start` 시 Task 분석 및 서브태스크 제안
- 복잡한 코드 분석 필요 시
- 의존성/영향도 분석 시
- 아키텍처 파악 시

## 호출 방법

```
Task tool 사용:
- subagent_type: "Plan" 또는 "Explore"
- model: "opus"
- prompt: 분석할 내용 명시
```

## 분석 유형

### Task 분석
```
"이 Task를 분석하고 서브태스크를 제안해주세요:
- Task: {task 이름}
- 목표: {목표}
- 레포: {사용 가능한 레포}"
```

### 코드 영향도 분석
```
"다음 변경의 영향도를 분석해주세요:
- 변경 파일: {파일 목록}
- 변경 내용: {내용 요약}"
```

### 의존성 분석
```
"다음 모듈의 의존성을 분석해주세요:
- 모듈: {모듈 경로}
- 관심사: {무엇을 알고 싶은지}"
```

## 반환 형식

```markdown
## 분석 결과

### 요약
{1-2문장 요약}

### 상세
{상세 분석 내용}

### 제안
{액션 아이템}
```

## 활용 내역 표시

```
🤖 Agent: analyzer (opus)
├─ 입력: Task 분석 요청
├─ 분석: 코드베이스 탐색, 의존성 파악
└─ 출력: 서브태스크 3개 제안
```
