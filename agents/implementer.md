# implementer: 구현 에이전트

---
model: sonnet
type: subagent
triggers: [구현, implement, 코딩, 작성, 생성]
---

코드 구현을 담당하는 서브에이전트입니다.

## 메타데이터

| 항목 | 값 |
|------|-----|
| **모델** | `sonnet` (균형잡힌 구현에 적합) |
| **컨텍스트** | 별도 (필요한 정보만 전달) |
| **용도** | 독립적인 구현 단위 위임 |

## 사용 시점

- 서브태스크 내 독립적인 구현 단위
- 보일러플레이트 코드 생성
- 정해진 패턴의 반복 구현
- 리팩토링 작업

## 호출 방법

```
Task tool 사용:
- subagent_type: "general-purpose"
- model: "sonnet"
- prompt: 구현할 내용 + 필요한 컨텍스트
```

## 전달할 정보

```markdown
## 구현 요청

### 목표
{무엇을 구현할지}

### 위치
{어느 파일에}

### 참고
{관련 코드, 패턴, 규칙}

### 제약
{지켜야 할 것}
```

## 반환 형식

```markdown
## 구현 완료

### 변경 파일
- {파일1}: {변경 내용}
- {파일2}: {변경 내용}

### 요약
{무엇을 구현했는지}

### 테스트 필요
{확인이 필요한 것}
```

## 주의사항

- 구현 범위를 명확히 제한
- 필요한 컨텍스트만 전달 (과다 전달 금지)
- 결과 검토 후 메인에서 커밋

## 활용 내역 표시

```
🤖 Agent: implementer (sonnet)
├─ 입력: PaymentService 구현 요청
├─ 작업: 3개 파일 생성/수정
└─ 출력: 구현 완료, 테스트 필요 사항 안내
```
