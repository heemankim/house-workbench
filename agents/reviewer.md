# reviewer: 리뷰 에이전트

---
model: haiku
type: subagent
triggers: [리뷰, review, 검토, 확인, 체크]
---

코드 리뷰를 담당하는 서브에이전트입니다.

## 메타데이터

| 항목 | 값 |
|------|-----|
| **모델** | `haiku` (빠른 검증에 적합) |
| **컨텍스트** | 별도 (리뷰 대상 코드만 전달) |
| **용도** | 객관적 시각으로 코드 검토 |

## 사용 시점

- `/wb-done` 전 셀프 리뷰
- PR 생성 전 사전 검토
- 복잡한 변경 검증
- 빠른 sanity check

## 호출 방법

```
Task tool 사용:
- subagent_type: "general-purpose"
- model: "haiku"
- prompt: 리뷰할 코드 + 체크리스트
```

## 리뷰 체크리스트

### 기능
- [ ] 요구사항 충족
- [ ] 엣지 케이스 처리
- [ ] 에러 핸들링

### 코드 품질
- [ ] 가독성
- [ ] 중복 제거
- [ ] 네이밍

### 보안
- [ ] 입력 검증
- [ ] 인증/인가
- [ ] 민감 정보 노출

### 성능
- [ ] 불필요한 연산
- [ ] N+1 쿼리
- [ ] 메모리 누수

## 전달할 정보

```markdown
## 리뷰 요청

### 변경 내용
{diff 또는 변경 파일 내용}

### 목적
{이 변경의 목적}

### 중점 검토
{특히 확인해야 할 부분}
```

## 반환 형식

```markdown
## 리뷰 결과

### 요약
✅ 승인 / ⚠️ 수정 필요 / ❌ 재작업 필요

### 이슈
1. [심각도] {이슈 내용}
   - 위치: {파일:라인}
   - 제안: {수정 방안}

### 잘된 점
- {칭찬할 부분}

### 제안
- {개선 제안}
```

## 활용 내역 표시

```
🤖 Agent: reviewer (haiku)
├─ 입력: 5개 파일 변경 리뷰 요청
├─ 검토: 체크리스트 기반 검증
└─ 출력: ⚠️ 수정 필요 (이슈 2개)
```
