# wb-done: 추가 서브태스크 유형 프로세스

## Wait 유형 프로세스 (⏳)

외부 의존이 있는 대기/협의 작업의 완료 처리입니다.

### 2w. 결과 확인

```
📋 대기 항목의 결과를 확인합니다:

원래 대기 내용:
  → 기획팀과 수수료율, 한도 정책 협의

결과:
[1] 결정됨 - 협의/결정 완료
[2] 보류 - 일부 결정, 추후 재논의 필요
[3] 취소 - 더 이상 필요 없음
```

### 3w. 결정 사항 기록

```
📝 결정된 내용을 기록하세요:

> 수수료율 2.5%, 건당 한도 100만원, 일일 한도 500만원
> 정책 문서: https://notion.so/xxx
```

### 4w. 의존 서브태스크 해제

```
🔓 이 완료로 blocked 해제되는 서브태스크:
  - #4. [📦 code] 결제 UI (frontend)
  - #6. [🔧 ops] 프로덕션 배포

해제하시겠습니까? (Y/n)
```

### 5w. 히스토리 업데이트

```markdown
### 서브태스크 완료: 정책 협의 - HH:MM

#### 결과
결정됨

#### 결정 사항
- 수수료율 2.5%
- 건당 한도 100만원
- 일일 한도 500만원

#### 참조
- 정책 문서: https://notion.so/xxx

#### Unblocked
- #4. 결제 UI
- #6. 프로덕션 배포
```

### 6w. 완료 메시지 (wait)

```
┌─────────────────────────────────────────────────────────┐
│ ✅ 서브태스크 완료: 정책 협의                            │
│ ⏳ 유형: wait                                           │
├─────────────────────────────────────────────────────────┤
│ 📋 결과: 결정됨                                         │
│ 📝 결정: 수수료율 2.5%, 건당 한도 100만원...            │
│ 🔓 Unblocked: 2개 서브태스크                           │
└─────────────────────────────────────────────────────────┘

다음 단계:
- /wb-next로 다음 서브태스크 (unblocked 포함)
```

## Research 유형 프로세스 (🔍)

### 2r. 조사 결과 유형

```
🔍 조사 결과 유형을 선택하세요:

[1] 기술 조사 - 라이브러리, 프레임워크, 도구 비교
[2] 문제 분석 - 버그, 성능 이슈 원인 파악
[3] 시장 조사 - 경쟁사, 트렌드 분석
[4] 요구사항 분석 - 기능 명세, 범위 정의
```

