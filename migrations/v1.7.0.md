# v1.7.0: wb-update 원스톱 업데이트

## 변경 요약

| 항목 | 이전 | 이후 |
|------|------|------|
| 업데이트 과정 | 3단계 수동 (git pull → cp → /wb-update) | `/wb-update` 1회 |
| 소스 동기화 | 사용자가 수동 cp -R | 자동 rsync (`.git` 제외) |
| 오프라인 대응 | 실패 시 수동 대응 | 자동 fallback (기존 캐시 사용) |
| 소스 경로 | 하드코딩 (~/.workbench/core) | config.json pluginSource로 변경 가능 |

### 배경

업데이트가 3단계로 분리되어 있어 사용자가 불편하다는 피드백.
`/wb-update`가 소스 pull부터 캐시 동기화, 마이그레이션까지 모두 처리하도록 통합.

## 자동 마이그레이션

이 버전에는 자동 마이그레이션이 필요하지 않습니다.
wb-update 스킬 자체가 업데이트되어 다음 실행부터 자동 적용됩니다.

## 수동 확인

선택사항: 기본 소스 경로가 `~/.workbench/core`가 아닌 경우,
config.json에 `pluginSource`를 설정하세요:

```json
{
  "pluginSource": "/your/custom/path/to/workbench"
}
```

## What's New

```
원스톱 업데이트
├─ /wb-update 한 번으로 소스 pull + 캐시 동기화 + 마이그레이션
├─ 수동 git pull, cp -R 불필요
├─ .git 폴더 자동 제외 (권한 문제 방지)
├─ 오프라인 시 기존 캐시로 자동 fallback
└─ config.json pluginSource로 소스 경로 커스터마이징 가능
```
