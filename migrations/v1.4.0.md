# v1.4.0: Obsidian PARA 통합

## 변경 요약

| 항목 | 이전 | 이후 |
|------|------|------|
| 기본 태스크 관리 | local (data/tasks/ TASK.md만) | Obsidian vault PARA 구조 (기본값) |
| 기본 AI 메모리 | local (.memory/) | Obsidian vault (3-Resources/) |
| config.json | `adapters.taskManager/aiMemory` 필수 | `vault` 필드만으로 동작 (어댑터 선택) |
| local 어댑터 | 존재 | **제거** (obsidian이 곧 local) |
| data/tasks TASK.md | 전체 상태 관리 | 경량 포인터 (vault = SSoT) |
| 서브태스크 관리 | TASK.md 체크리스트만 | SUBTASK.md 작업지시서 (분석, 체크리스트, 의사결정, 히스토리) |
| 프로젝트 구조 | 플랫 TASK.md | TASK.md + subtasks/ + decisions.md + notes/ |
| wb-resume/save | 미정의 | 구체적 프로시저 정의 |
| wb-cancel | 없음 | 태스크 취소 + 서브태스크 건너뛰기 |
| 서브태스크 상태 | 2종 (완료/미완료) | 6종 (pending/in_progress/done/blocked/skipped/cancelled) |

### 배경

컨텍스트 엔지니어링에서 로컬 파일 직접 Read가 API 호출보다 빠르고 정확.
Obsidian vault에 PARA 방법론을 적용하여 태스크 관리 + 지식 관리를 통합.

## 자동 마이그레이션

### Step 1: config.json 업데이트

**vault 필드 추가** (obsidian을 사용하는 경우):
```json
{
  "vault": "obsidian",
  "pluginVersion": "1.4.0"
}
```

**기존 adapters 블록 정리:**
- `adapters.taskManager.type: "local"` → 제거 (vault 필드로 대체)
- `adapters.aiMemory.type: "local"` → 제거 (vault 필드로 대체)
- todoist/tududi/graphiti 등 외부 어댑터는 유지

> vault 필드가 있으면 adapters.taskManager와 adapters.aiMemory는 무시됩니다.
> 외부 어댑터(todoist, graphiti 등)를 사용 중이면 vault 필드를 추가하지 마세요.

### Step 2: Obsidian vault 초기화 (vault 선택 시)

```bash
# PARA 폴더 구조 생성
mkdir -p {vault}/{0-Inbox,1-Projects,2-Areas,3-Resources/{patterns,troubleshooting,guides,references},4-Archives,_templates,daily,weekly}
```

### Step 3: 기존 활성 태스크 vault 이동 (선택)

기존 `data/tasks/`의 활성 태스크를 vault로 이동하려면:

```
각 활성 태스크에 대해:
1. {vault}/1-Projects/{task-name}/ 폴더 생성
2. {vault}/1-Projects/{task-name}/subtasks/ 폴더 생성
3. {vault}/1-Projects/{task-name}/notes/ 폴더 생성
4. data/tasks/{task-name}/TASK.md 내용으로 vault TASK.md 생성 (frontmatter 변환)
5. decisions.md 초기화
6. data/tasks/{task-name}/TASK.md를 경량 포인터로 변환:
   vaultPath: "1-Projects/{task-name}"
```

> 이 단계는 선택입니다. 기존 태스크는 data/tasks/에서 계속 관리해도 됩니다.
> 새로 시작하는 태스크부터 vault 모드가 적용됩니다.

### Step 4: 기존 완료 태스크 → Archives (선택)

`data/history/`의 완료된 태스크를 vault Archives로 이동:

```
각 완료 태스크에 대해:
1. {vault}/4-Archives/{task-name}/ 폴더 생성
2. data/history/{task-name}.md 내용으로 TASK.md 생성
```

### Step 5: 템플릿 생성

vault에 `_templates/` 폴더가 없으면 6개 기본 템플릿을 생성합니다:
- tpl-project.md, tpl-subtask.md, tpl-daily.md, tpl-weekly.md, tpl-decision.md, tpl-resource.md

### Step 6: CLAUDE.md 업데이트 (선택)

Workspace CLAUDE.md에 vault 정보를 반영:
```markdown
## 설정
- Vault: obsidian/ (Obsidian PARA)
- Task Manager: obsidian (1-Projects/, 4-Archives/)
- AI Memory: obsidian (3-Resources/, 2-Areas/)
```

### Step 7: pluginVersion 업데이트

```json
{
  "pluginVersion": "1.4.0"
}
```

## What's New

### Obsidian PARA 통합 (기본값)
- `config.json`에 `"vault": "obsidian"` 추가만으로 Obsidian 기반 태스크/지식 관리 활성화
- PARA 방법론: Projects(활성) → Archives(완료), Resources(지식), Areas(영역)
- Obsidian GUI + Claude Code 양쪽에서 동일 데이터 접근
- 외부 API 없이 로컬 파일시스템만 사용 (빠른 컨텍스트 로딩)

### 서브태스크 작업지시서 (SUBTASK.md)
- 서브태스크별 독립 파일: 목표, 분석, 체크리스트, 의사결정, 히스토리
- `wb-next` 시 subtask-analyzer가 자동 분석
- `wb-resume` 시 SUBTASK.md 하나로 컨텍스트 복원 (3~4회 Read)

### 신규 스킬 (6개)
- `obsidian-task-adapter` — 파일시스템 기반 PARA 태스크 관리
- `obsidian-memory-adapter` — 3단계 검색 파이프라인 지식 관리
- `subtask-analyzer` — 서브태스크 사전 분석 → SUBTASK.md 생성
- `decision-logger` — 의사결정 자동 감지/기록
- `weekly-review` — 주간 리뷰 자동 생성 (PARA 메인터넌스 포함)
- `knowledge-extract` — 태스크 완료 시 재사용 지식 추출

### 신규 커맨드
- `/wb-save` — 중간 저장 (체크리스트 + 히스토리 + 의사결정 확인)
- `/wb-resume` — 컨텍스트 복원 (vault TASK.md + SUBTASK.md + decisions.md)
- `/wb-cancel` — 태스크 취소 또는 서브태스크 건너뛰기
- `/wb-weekly-review` — 주간 리뷰 생성

### 개선된 기능
- 서브태스크 6종 상태: pending, in_progress, done, blocked, skipped, cancelled
- 의존성 관리: `wb-next`에서 depends 확인 + blocked 자동 관리
- 스킬 의존성 사전 로드 맵 (AI 컨텍스트 최적화)
- data/tasks TASK.md → 경량 포인터 (vault = Single Source of Truth)
- 검색 3단계 파이프라인: tags → TL;DR → 본문 전체
- Archives 재활용: wb-start 시 유사 완료 태스크 검색

### local 모드 제거
- `adapters.taskManager.type: "local"` / `adapters.aiMemory.type: "local"` 제거
- Obsidian vault 자체가 로컬 파일시스템이므로 별도 local 모드 불필요
- 외부 어댑터(todoist, tududi, graphiti, serena)는 계속 지원

### 영향
- 기존 외부 어댑터 사용자: 변경 없음 (vault 필드 미설정 시 기존 동작 유지)
- 신규/local 사용자: `vault` 필드 추가 권장 (더 풍부한 태스크/지식 관리)
- `/wb-init`: obsidian이 기본 선택지, PARA 폴더 자동 생성
