# v1.8.3: Vault 검색 스킬 FTS5/grep config 분기

## 변경 요약

| 항목 | 이전 | 이후 |
|------|------|------|
| vault 검색 방식 | grep 파이프라인 하드코딩 | config.json `beta.fts-indexer` 분기 |
| 영향 스킬 | 5개 (wb-start, wb-next, subtask-analyzer, wb-recall, wb-refresh) | 동일 |

### 배경

vault-conventions에 FTS5/grep 두 모드를 정의해두었으나, 실제 소비자 스킬들이 모두 grep 모드만 하드코딩하고 있어 `beta.fts-indexer: true` 설정이 무시되던 문제 수정.

## 자동 마이그레이션

없음. SKILL.md 파일 변경만 포함되며, 캐시 동기화로 자동 적용됩니다.

## 수동 확인

FTS5를 사용하려면 config.json에 다음이 설정되어 있어야 합니다:

```json
{
  "beta": {
    "fts-indexer": true
  }
}
```

미설정 시 기존 grep 파이프라인이 그대로 동작합니다.

## What's New

```
Vault 검색 FTS5 실제 통합
├─ wb-start, wb-next, subtask-analyzer, wb-refresh: 2-stage grep → config 분기
├─ wb-recall: 3-stage grep → FTS5 단일 검색 / grep 3-stage 분기
├─ FTS5 모드: python3 tools/vault-search.py search 사용
├─ grep 모드: 기존 multi-stage 파이프라인 유지
└─ config.json beta.fts-indexer 값으로 자동 전환
```
