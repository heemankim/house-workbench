# v1.8.1: FTS5 TL;DR 원본 텍스트 유지

## 변경 요약

| 항목 | 이전 | 이후 |
|------|------|------|
| 검색 결과 tldr/tags | 토크나이즈된 텍스트 | 원본 텍스트 |
| DB schema | v2 | v3 (자동 재구축) |
| FTS5 컬럼 | 7개 | 9개 (+tags_raw, tldr_raw UNINDEXED) |

### 배경

v1.8.0에서 kiwipiepy pre-tokenize 적용 후, tldr/tags가 토크나이즈된 형태로 저장되어
검색 결과에서 원본이 아닌 분리된 텍스트가 노출되는 문제 수정 (#17).

## 자동 마이그레이션

### 1. tools/vault-search.py 업데이트

플러그인 캐시의 최신 vault-search.py를 workspace tools/에 복사:

```bash
cp {plugin-cache}/tools/vault-search.py tools/vault-search.py
```

### 2. DB 자동 재구축

schema_version이 v2이면 다음 검색/인덱싱 시 자동으로 DB를 재구축합니다.
수동 재인덱싱도 가능:

```bash
python3 tools/vault-search.py index --vault {vault-path}
```

## 수동 확인

`python3 tools/vault-search.py status`로 확인:

```json
{
  "schema_version": "3",
  "korean_tokenizer": true
}
```

- `schema_version: "3"` — 원본 텍스트 보존 스키마 적용됨

## What's New

```
FTS5 TL;DR 원본 텍스트 유지 (#17)
├─ 검색 결과에서 토크나이즈된 텍스트 대신 원본 텍스트 반환

```
