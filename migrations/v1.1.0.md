# v1.1.0: Skills 폴더 경로 통일

## 변경 요약

Workspace skills 폴더 위치를 `.claude/skills/`로 통일합니다.

| 항목 | 이전 (v1.0.0) | 이후 (v1.1.0) |
|------|--------------|--------------|
| Workspace skills 경로 | `skills/` | `.claude/skills/` |
| Task Manager 오버라이드 | `skills/task-manager.md` | `.claude/skills/task-manager.md` |
| 스킬 저장 위치 (/wb-save-skill) | `skills/{name}.md` | `.claude/skills/{name}.md` |

### 배경

Claude Code는 `.claude/skills/`를 네이티브 경로로 사용합니다.
기존에는 Workspace 루트의 `skills/`를 사용했으나, Claude Code 네이티브 경로와 일치시켜 혼란을 제거합니다.

## 자동 마이그레이션

다음 단계를 순서대로 실행합니다:

### Step 1: skills/ 폴더 이동

```
조건: skills/ 폴더가 워크스페이스 루트에 존재하는 경우

1. .claude/skills/ 폴더가 없으면 생성
2. skills/ 내 파일을 .claude/skills/로 이동 (mv skills/* .claude/skills/)
3. 빈 skills/ 폴더 삭제 (rmdir skills/)

조건: skills/ 폴더가 없는 경우
→ .claude/skills/ 폴더만 생성 (mkdir -p .claude/skills/)
```

### Step 2: CLAUDE.md 경로 업데이트

```
Workspace CLAUDE.md에서 다음 패턴을 찾아 교체:

1. "skills/task-manager.md" → ".claude/skills/task-manager.md"
2. "skills/에 추가 가능" → ".claude/skills/에 추가 가능"
3. 구조 다이어그램에서 "├── skills/" → ".claude/skills/" 반영
   (다이어그램은 수동 확인 권장)
```

### Step 3: config.json pluginVersion 추가

```
config.json에 "pluginVersion": "1.1.0" 추가
```

## 수동 확인

- Workspace CLAUDE.md의 구조 다이어그램이 올바른지 확인
- 기존에 skills/ 내 커스텀 스킬이 있었다면 .claude/skills/로 잘 이동되었는지 확인

## What's New

```
Skills 경로 표준화
├─ Workspace skills가 .claude/skills/로 통일되었습니다
├─ Claude Code 네이티브 경로와 일치합니다
└─ 기존 skills/ 파일들은 자동 이동되었습니다

영향:
├─ /wb-start: .claude/skills/task-manager.md를 참조합니다
├─ /wb-save-skill: .claude/skills/에 저장합니다
└─ 새 스킬 추가 시 .claude/skills/에 생성하세요
```
