# v1.9.0: Session Start Vault SSoT + wb-update 캐시 수정

## 변경 요약

| 항목 | 이전 | 이후 |
|------|------|------|
| session-start 태스크 소스 | data/tasks/TASK.md만 파싱 | vault TASK.md 우선 (SSoT) |
| 서브태스크 파싱 | 체크박스만 (`- [x] 01.`) | 체크박스 + 테이블 형식 |
| 완료 태스크 필터링 | data/tasks 기준 | vault 기준 (정확한 진행률) |
| vault 경로 | 하드코딩 없음 | config.json `vault` 필드 사용 |
| wb-update 캐시 동기화 | rsync (샌드박스 차단) | find + cp 패턴 |
| wb-start 서브태스크 확인 | 마크다운 출력 (UI에서 안 보임) | AskUserQuestion preview 사용 |

### 배경

session-start.js의 `scanIncompleteTasks()`가 data/tasks/TASK.md만 파싱하여 vault와 진행률 불일치 발생.
완료된 태스크가 미완료로 표시되거나, 서브태스크 수가 다른 문제.

## 자동 마이그레이션

없음. 캐시 동기화(`/wb-update`)로 자동 적용됩니다.

## 수동 확인

config.json에 `vault` 필드가 설정되어 있어야 합니다:

```json
{
  "vault": "obsidian"
}
```

미설정 시 session-start가 data/tasks/TASK.md로 fallback하므로 기존과 동일하게 동작합니다.

## What's New

```
Vault SSoT + wb-update 수정
├─ session-start: vault TASK.md를 SSoT로 우선 읽기
│   ├─ vaultPath → fallback 1-Projects/{taskName} → data/tasks 순서
│   ├─ 체크박스 + 테이블 서브태스크 파싱
│   ├─ config.json vault 필드로 경로 동적 결정
│   └─ 완료 태스크 정확한 필터링
├─ wb-start: 서브태스크 초안을 AskUserQuestion preview에 포함
└─ wb-update: rsync → find+cp (샌드박스 호환)
```
