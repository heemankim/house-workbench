# v1.2.0 마이그레이션

## 변경 사항

### 1. commands/ → skills/ 통합

`commands/` 디렉토리가 `skills/`로 통합되었습니다.
모든 wb-* 커맨드 파일과 내부 스킬 파일이 하나의 `skills/` 디렉토리에서 관리됩니다.

**이전 구조:**
```
commands/          ← wb-* 커맨드 (사용자 호출용)
skills/            ← 내부 스킬 (커맨드에서 참조)
```

**새 구조:**
```
skills/            ← 모든 스킬 통합 (커맨드 + 내부 스킬)
```

### 2. TASK.md frontmatter 표준화

`type` 파일이 제거되고 TASK.md frontmatter로 통합되었습니다.
외부 Task Manager ID도 frontmatter에서 관리합니다.

**이전:**
```
data/tasks/{task}/
├── type: code          ← 별도 파일
├── TASK.md             ← frontmatter 없음
```

**이후:**
```
data/tasks/{task}/
├── TASK.md             ← frontmatter에 type + taskManager ID 포함
```

### 3. Skills 폴더 구조 전환

스킬 파일이 플랫 구조에서 폴더 구조로 변경되었습니다.

**이전:** `skills/{name}.md`
**이후:** `skills/{name}/SKILL.md`

이 구조는 Claude Code의 권장 패턴을 따르며, 스킬당 추가 파일(스크립트, 예시 등)을 포함할 수 있습니다.
`session-start.js`는 폴더 구조와 레거시 플랫 구조 모두 자동 감지합니다.

### 4. worktree-guard 스킬 추가

`repos/` 하위 코드 직접 수정 시도를 감지하고 worktree 사용을 안내하는 가드 스킬이 추가되었습니다.
CLAUDE.md "항상 활성 규칙"에 등록되어 모든 세션에서 자동 적용됩니다.

### 5. Hook 시스템 수정

- `session-start.js`: `.claude/skills/` 우선 스캔 + `skills/` fallback
- `detect-decision.sh`: JSON 입력에서 `new_string`/`content` 추출 후 패턴 검색 (기존: 원시 JSON 검색)
- `README.md` hooks 섹션: 실제 `hooks.json` 기준으로 재작성 (`chain.sh` 제거)

### 4. DRY 개선

- `wb-done`, `wb-add-subtask`, `wb-start`, `wb-end`: 인라인 Task Manager 동기화 블록 → `task-manager.md` 참조로 교체
- `wb-memo`: Graphiti/Serena API 호출 → `memory-manager.md` Save 참조로 교체

### 5. 안전성 개선

- `wb-end`: `rm -rf` 전 히스토리 저장 확인 + 사용자 확인 단계 추가
- `task-manager.md`: `config.json` 파일 부재 시 local fallback 명시

### 6. 지식 승격 제안 (wb-end)

Task 완료 시 컨텍스트 분석을 통해 재사용 가능한 지식을 제안합니다:
- **Skill 생성**: 재사용 가능한 패턴/가이드 (예: "Stripe 연동 가이드")
- **Command 생성**: 반복 워크플로우 (예: "/deploy-staging")
- **기존 Skill 업데이트**: 기존 스킬에 새 정보 추가

### 7. 문서 일관성

- `CLAUDE.md`, `templates/workspace/CLAUDE.md`: `checker` 에이전트 추가
- `wb-start.md`: 완료 메시지 경로 `data/tasks/` 통일
- README.md 경로: `tasks/` → `data/tasks/` 통일
- wb-memo/wb-recall: Task 컨텍스트 자동 연결 추가
- wb-run: 자동 wb-done 처리 + 배치 PR 옵션 추가
- wb-plan/wb-start: wb-run 파이프라인 연결 안내 추가
- wb-resume: dirty worktree 복구 로직 추가
- Serena 활성화 로직: code-analysis.md 스킬로 통합
- Task Manager: 외부 동기화 실패 시 fallback 정책 추가

## 자동 마이그레이션

### Workspace 커스텀 commands 이동

Workspace에 `.claude/commands/`가 있으면 `.claude/skills/`로 이동 (폴더 구조):

```bash
# commands/ → skills/ 이동 + 폴더 구조 변환
if [ -d ".claude/commands" ]; then
    for f in .claude/commands/*.md; do
        [ -f "$f" ] || continue
        name=$(basename "$f" .md)
        mkdir -p ".claude/skills/$name"
        mv "$f" ".claude/skills/$name/SKILL.md"
    done
    rmdir .claude/commands 2>/dev/null
fi
```

### 기존 플랫 skills 파일을 폴더 구조로 변환

```bash
# skills/{name}.md → skills/{name}/SKILL.md
if [ -d ".claude/skills" ]; then
    for f in .claude/skills/*.md; do
        [ -f "$f" ] || continue
        name=$(basename "$f" .md)
        mkdir -p ".claude/skills/$name"
        mv "$f" ".claude/skills/$name/SKILL.md"
    done
fi
```

### Workspace CLAUDE.md 업데이트

CLAUDE.md에서 `commands/` 참조를 `skills/`로 변경:

```
- "commands/" → 제거 또는 "skills/"로 변경
- Workspace 구조 다이어그램 업데이트
```

### 기존 Task의 type 파일 정리

진행 중인 Task가 있으면:

```bash
# 각 Task 폴더에서 type 파일 확인
for task_dir in data/tasks/*/; do
    if [ -f "$task_dir/type" ]; then
        TYPE=$(cat "$task_dir/type" | tr -d '[:space:]')
        # TASK.md frontmatter에 type이 없으면 추가
        # type 파일 삭제
        rm "$task_dir/type"
    fi
done
```

### config.json pluginVersion 업데이트

```json
{
  "pluginVersion": "1.2.0"
}
```

## What's New

### commands → skills 통합 + 폴더 구조
- 모든 파일이 `skills/` 하나로 통합되었습니다
- 스킬 파일이 `skills/{name}/SKILL.md` 폴더 구조로 변경되었습니다
- plugin.json에서 `commands` 필드가 제거되었습니다
- 기존 `/wb-*` 호출 방식은 변경 없습니다

### TASK.md frontmatter
- Task 유형(`type`)이 별도 파일 대신 TASK.md frontmatter에 저장됩니다
- 외부 Task Manager ID도 frontmatter에서 관리됩니다
- 외부 동기화 실패 시 TASK.md fallback 정책이 추가되었습니다

### 워크플로우 개선
- `/wb-run`: 완료된 서브태스크 자동 `wb-done` 처리, 배치 PR 옵션
- `/wb-plan` → `/wb-start` → `/wb-run` 파이프라인 안내
- `/wb-resume`: dirty worktree 자동 감지 및 복구 옵션
- `/wb-memo`, `/wb-recall`: Task 컨텍스트 자동 연결

### Hook 시스템 정비
- `session-start.js`: `.claude/skills/` 우선 스캔으로 변경
- `detect-decision.sh`: 실제 코드 변경 내용에서 패턴 검색하도록 수정
- README hooks 섹션: 실제 구성(SessionStart/PostToolUse/Stop)으로 재작성

### 지식 승격 제안
- `/wb-end` 완료 시 Task 컨텍스트를 분석하여 Skill/Command 생성 또는 기존 Skill 업데이트를 제안합니다

### DRY 및 안전성
- Task Manager 동기화 로직: 인라인 분기 → `task-manager.md` 단일 참조
- Memory 저장 로직: 인라인 API 호출 → `memory-manager.md` 참조
