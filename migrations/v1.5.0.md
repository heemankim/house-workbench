# v1.5.0: 컨텍스트 최적화

## 변경 요약

| 항목 | 이전 (v1.4.x) | 이후 (v1.5.0) |
|------|---------------|---------------|
| Vault 규칙 | 5개 어댑터 스킬에 분산 | `vault-conventions` 1개로 통합 |
| 어댑터 스킬 | task-manager, obsidian-task-adapter, memory-manager, obsidian-memory-adapter, serena-adapter | **전체 삭제** |
| "AI Memory" 개념 | memory-manager → adapter 라우팅 | 제거 (vault 직접 I/O) |
| Vault 검색 | 매 커맨드마다 검색 | 프론트로딩 (wb-start/wb-next에서 임베딩) |
| TASK.md | taskManager 블록 포함 | lastScanned + `## 관련 지식` 섹션 |
| SUBTASK.md | 분석만 포함 | `## 관련 지식` 임베딩 포함 |
| wb-resume | vault 재검색 (29.9KB) | 문서만 읽기 (6.8KB, -77%) |
| subtask-analyzer | 무제한 탐색 | 범위 제한 (50파일, 제외 경로) |
| wb-refresh | 없음 | **신규** (프론트로딩 갱신) |

### 배경

문서량/태스크 증가에 따라 컨텍스트 소비가 급격히 증가.
어댑터 체인 간접비(~27KB/커맨드)와 매번 vault 검색이 주요 병목.
프론트로딩(Scan Once, Read Many) 모델로 전환하여 세션 컨텍스트 ~61% 절감.

## 자동 마이그레이션

### Step 1: config.json 정리

**adapters 블록 제거** (이미 vault 모드면):
```json
// 제거 대상
"adapters": {
  "taskManager": { "type": "obsidian" },
  "aiMemory": { "type": "obsidian" }
}

// 유지
{
  "vault": "obsidian",
  "pluginVersion": "1.5.0"
}
```

> vault 필드가 있으면 adapters 블록은 완전히 불필요합니다.
> 외부 어댑터(todoist 등)를 사용하고 있다면 해당 블록만 유지하세요.

### Step 2: CLAUDE.md 업데이트

Workspace CLAUDE.md에서 "AI Memory" 관련 텍스트를 정리:
```markdown
## 설정
- Vault: obsidian/ (Obsidian PARA)
- Task Manager: obsidian (1-Projects/, 4-Archives/)
- AI Memory: obsidian (3-Resources/, 2-Areas/)  ← 이 줄 제거

// 변경 후
## 설정
- Vault: obsidian/ (Obsidian PARA)
- Task Manager: obsidian (1-Projects/, 4-Archives/)
```

### Step 3: 기존 활성 TASK.md frontmatter 갱신 (선택)

기존 vault TASK.md에 `lastScanned` 필드가 없으면 추가:
```yaml
---
type: code
created: 2026-03-01
lastScanned: ""    # 다음 wb-resume 시 자동 채워짐
---
```

`taskManager` 블록이 있으면 제거:
```yaml
# 제거
taskManager:
  adapter: "obsidian"
  taskId: "..."
  subtasks: {}
```

> 이 단계는 선택입니다. lastScanned가 없으면 wb-resume이 staleness 체크를 건너뜁니다.

### Step 4: pluginVersion 업데이트

```json
{
  "pluginVersion": "1.5.0"
}
```

## What's New

### 컨텍스트 최적화 (-61%)
- 어댑터 체인 제거: 5개 스킬(28.8KB) → vault-conventions 1개(3.3KB)
- 프론트로딩: wb-start/wb-next에서 vault 검색 후 TASK.md/SUBTASK.md에 임베딩
- wb-resume이 문서만 읽어 컨텍스트 복원 (vault 재검색 없음)
- 세션 총 컨텍스트: ~570KB → ~222KB

### "AI Memory" 개념 제거
- memory-manager, obsidian-memory-adapter 삭제
- vault 파일 직접 Read/Write로 단순화
- wb-memo: vault에 직접 저장 (어댑터 라우팅 제거)
- wb-recall: vault-conventions 2-stage 검색

### 프론트로딩 (Scan Once, Read Many)
- wb-start: vault 검색 → TASK.md `## 관련 지식`에 임베딩
- wb-next: subtask-analyzer → SUBTASK.md `## 관련 지식`에 임베딩
- wb-resume: 임베딩된 지식만 읽음 (vault 검색 0회)
- lastScanned: TASK.md frontmatter에 타임스탬프 기록

### staleness 감지 + /wb-refresh
- wb-resume 시 `find -newer` (~0 비용)로 vault 변경 감지
- 변경 있으면 `/wb-refresh` 안내 (강제하지 않음)
- `/wb-refresh`: 관련 지식 재검색 + TASK.md/SUBTASK.md 갱신

### subtask-analyzer 범위 제한
- git diff 우선 분석
- 최대 50개 파일 탐색
- node_modules, dist, build, vendor 자동 제외
- 결과 크기 제한 (경로+라인 범위만, 코드 본문 X)

### vault-conventions 통합
- PARA 경로 매핑, TASK.md/SUBTASK.md 스키마, 검색 파이프라인을 하나로
- 2-stage 검색: frontmatter tags → TL;DR 섹션
- Stage 3 (전체 vault grep)는 wb-recall 전용
- data/tasks 포인터 형식 통일

### 삭제된 스킬 (5개)
- `task-manager` — vault-conventions으로 통합
- `obsidian-task-adapter` — vault-conventions으로 통합
- `memory-manager` — 개념 자체 제거
- `obsidian-memory-adapter` — vault-conventions으로 통합
- `serena-adapter` — 별도 스킬 불필요 (code-analysis가 직접 사용)

### 신규 스킬
- `wb-refresh` — 프론트로딩된 지식 갱신
- `vault-conventions` — vault 규칙 통합 레퍼런스

### 영향
- 기존 워크스페이스: wb-update 실행으로 자동 마이그레이션
- 커스텀 스킬: task-manager/memory-manager 참조 → vault-conventions으로 변경 필요
- 외부 어댑터 사용자: 변경 없음 (adapters 블록은 유지 가능)
