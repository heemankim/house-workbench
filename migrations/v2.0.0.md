# v2.0.0: SUBTASK.md 가드레일 — 누락 방지 및 공통 템플릿

## 변경 요약

| 항목 | 이전 | 이후 |
|------|------|------|
| SUBTASK.md 템플릿 | subtask-analyzer에 인라인 | vault-conventions에 공통 정의 |
| wb-start 생성 범위 | 첫 서브태스크만 | **모든** 서브태스크 |
| wb-add-subtask | 빈 폴더만 생성 | 폴더 + SUBTASK.md 원자적 생성 |
| wb-next | SUBTASK.md 존재 가정 | 존재 체크 + fallback 생성 |
| wb-done | SUBTASK.md 존재 가정 | 사후 가드레일 (미존재 시 자동 생성) |
| 플러그인 버전 | 1.9.1 | 2.0.0 |

### 배경

wb-next를 거치지 않고 직접 서브태스크를 진행하면, 디렉토리만 있고 SUBTASK.md가 없는 상태가 발생.
wb-done이 상태 업데이트를 시도할 때 대상 파일이 없어 불완전한 기록이 남는 문제.

### Breaking Changes

- **vault-conventions**: SUBTASK.md 최소 템플릿 섹션 추가. 이 섹션을 참조하는 모든 스킬이 새 규칙을 따름.
- **wb-add-subtask**: 더 이상 빈 폴더를 생성하지 않음. SUBTASK.md 동시 생성 필수.
- **subtask-analyzer**: 인라인 템플릿 제거. vault-conventions 참조로 전환.

## 자동 마이그레이션

없음. 캐시 동기화(`/wb-update`)로 스킬 파일이 자동 적용됩니다.

## 수동 확인

### 기존 고아 폴더 백필 (선택)

기존 태스크에 SUBTASK.md 없는 서브태스크 디렉토리가 있으면 수동 백필이 필요합니다:

```bash
# 고아 폴더 스캔 (SUBTASK.md 없는 서브태스크 디렉토리 찾기)
find data/tasks/*/subtasks/ -mindepth 1 -maxdepth 1 -type d ! -exec test -f {}/SUBTASK.md \; -print
```

발견된 폴더에 대해:
1. 해당 태스크의 TASK.md에서 서브태스크 이름과 유형을 확인
2. vault-conventions의 최소 템플릿으로 SUBTASK.md를 수동 생성
3. 이미 완료된 서브태스크라면 `status: done`, `completed` 날짜 설정
4. 진행 중인 서브태스크라면 `status: in_progress`, `started` 날짜 설정

> 진행 중인 태스크가 없으면 이 단계는 건너뛰어도 됩니다.
