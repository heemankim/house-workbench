# v2.1.0: 품질 게이트, 모듈화, FTS5 자동 인덱싱

## 변경 요약

| 항목 | 이전 | 이후 |
|------|------|------|
| wb-done (code) | 바로 완료 처리 | **자동 wb-check 게이트** + Release 체크리스트 |
| wb-check | checker만 호출 | checker + **tester agent 통합** |
| auto-save.sh | 히스토리 기록만 | 히스토리 + **FTS5 비동기 인덱싱** |
| vault-conventions TASK.md | type, created | type, created, **tags** (optional) |
| session-start.js | 단일 파일 (423 LOC) | hooks/lib/ 5개 모듈 (37 LOC 진입점) |
| Vault-First SSoT | data/tasks 중심 생성 | **vault 먼저 생성, data/tasks는 작업 공간만** |
| 플러그인 버전 | 2.0.1 | 2.1.0 |

### 배경

워크벤치 워크스페이스 종합 분석에서 다음 문제를 발견:
- wb-done 후 README/마이그레이션/버전 누락 반복 발생
- wb-check에서 테스트 커버리지 검증 누락
- FTS5 인덱스 수동 갱신 필요
- session-start.js 유지보수 어려움 (단일 파일)
- TASK.md tags 필드가 스키마에 미정의
- data/tasks/에 Task 상태가 중복 저장되어 vault SSoT 동기화 누락 반복 발생

### Breaking Changes

없음. 모든 변경은 하위 호환됩니다:
- wb-done 게이트: `--skip-check`로 우회 가능
- FTS5 인덱싱: `beta.fts-indexer: true` 조건부 실행
- tags 필드: optional (기존 문서 영향 없음)
- session-start.js: 출력 형식 100% 동일

### Vault-First SSoT 강화

5개 스킬에 vault-first 원칙을 명시적으로 적용:
- **wb-start**: vault TASK.md + SUBTASK.md 먼저 생성, data/tasks/는 작업 공간만
- **wb-done**: vault SUBTASK.md 상태 먼저 업데이트
- **wb-next**: vault SUBTASK.md를 SSoT로 읽기
- **wb-add-subtask**: vault에 먼저 추가, data/tasks/ 미러링
- **wb-save**: 의사결정은 vault에 필수 저장 (선택 아님)

**data/tasks/ 역할 재정의:**
```
vault (SSoT):              data/tasks/ (작업 공간):
├─ TASK.md                 ├─ worktree/
├─ subtasks/SUBTASK.md     ├─ history/
├─ decisions.md            └─ .claude → symlink
└─ CLAUDE.md (task)
```

## 자동 마이그레이션

캐시 동기화(`/wb-update`)로 스킬 파일이 자동 적용됩니다.

## 수동 확인

### 1. wb-done 게이트 인지

v2.1.0부터 code 타입 `/wb-done` 실행 시 자동으로 `/wb-check`가 호출됩니다:
- Match Rate >= 90%: 자동 통과
- Match Rate < 90%: 확인 프롬프트 표시
- 우회: `/wb-done --skip-check`

### 2. Release 체크리스트

code 타입 서브태스크 완료 시 다음을 자동 확인합니다:
- README 갱신 필요 여부
- 마이그레이션 파일 필요 여부
- 버전 범프 필요 여부
