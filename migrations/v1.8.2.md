# v1.8.2: FTS5 모든 사용자 노출 필드 원본 텍스트 유지

## 변경 요약

| 항목 | 이전 | 이후 |
|------|------|------|
| title 검색 결과 | 토크나이즈됨 ("종합 - 상담 - 결과 서") | 원본 ("종합-상담-결과서") |
| DB schema | v3 | v4 (자동 재구축) |
| FTS5 컬럼 | 9개 | 10개 (+title_raw UNINDEXED) |

### 배경

v1.8.1에서 tldr/tags 원본 보존을 추가했으나, title 필드도 동일하게 토크나이즈된 형태로 노출되는 문제 수정.

## 자동 마이그레이션

### 1. tools/vault-search.py 업데이트

```bash
cp {plugin-cache}/tools/vault-search.py tools/vault-search.py
```

### 2. DB 자동 재구축

schema_version이 v3이면 다음 검색/인덱싱 시 자동으로 DB를 재구축합니다.

```bash
python3 tools/vault-search.py index --vault {vault-path}
```

## 수동 확인

`python3 tools/vault-search.py status`로 확인:

```json
{
  "schema_version": "4",
  "korean_tokenizer": true
}
```

## What's New

```
FTS5 title 원본 텍스트 유지
├─ 검색 결과 title에서 토크나이즈된 텍스트 대신 원본 반환
├─ UNINDEXED 컬럼 추가 (title_raw)
├─ DB schema v4 (자동 마이그레이션, 재인덱싱)
└─ 이제 title, tldr, tags 모든 사용자 노출 필드가 원본 텍스트 유지
```
