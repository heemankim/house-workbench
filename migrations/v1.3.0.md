# v1.3.0: 어댑터 추상화

## 변경 요약

| 항목 | 이전 | 이후 |
|------|------|------|
| wb-* 커맨드 | 어댑터 직접 참조 (todoist, graphiti 등) | 추상 인터페이스 (task-manager, memory-manager) 참조 |
| memory-manager | Graphiti 전용 구현 | 추상 인터페이스 + 어댑터 라우팅 |
| task-manager | todoist/tududi 구현 인라인 | 추상 인터페이스 + 어댑터 라우팅 |
| 어댑터 스킬 | 없음 | graphiti-adapter, serena-adapter, todoist-adapter, tududi-adapter |

### 배경

워크플로우 커맨드(wb-*)가 특정 어댑터(todoist, graphiti)를 직접 참조하고 있어,
새 어댑터 추가 시 모든 커맨드를 수정해야 했음.
추상 인터페이스를 통한 접근으로 전환하여 어댑터 스킬만 추가하면 되도록 변경.

## 자동 마이그레이션

### Step 1: 플러그인 캐시 갱신

플러그인 소스가 업데이트되었으므로 캐시를 갱신합니다.
(이 마이그레이션 적용 시점에 이미 갱신된 상태)

### Step 2: config.json pluginVersion 업데이트

```json
{
  "pluginVersion": "1.3.0"
}
```

## 수동 확인

- Workspace CLAUDE.md에 어댑터 직접 참조가 남아있는지 확인
- `.claude/skills/`에 커스텀 커맨드가 어댑터를 직접 참조하고 있다면 memory-manager/task-manager 위임으로 변경

---

## What's New

### 어댑터 추상화
- wb-* 커맨드가 더 이상 특정 어댑터를 직접 참조하지 않습니다
- `config.json`의 `adapters.taskManager.type`과 `adapters.aiMemory.type`에 따라 자동 라우팅됩니다
- 새 어댑터 추가 시 커맨드 수정 없이 어댑터 스킬만 추가하면 됩니다

### 신규 어댑터 스킬 (4개)
- `graphiti-adapter` — Graphiti MCP 구현체
- `serena-adapter` — Serena Memory 구현체
- `todoist-adapter` — Todoist MCP 구현체
- `tududi-adapter` — Tududi REST API 구현체

### 매니저 인터페이스 개선
- `memory-manager` — Save/Search 프로시저가 어댑터 독립적으로 정의됨
- `task-manager` — 모든 wb-* 프로시저가 어댑터 독립적으로 정의됨

### 영향
- `/wb-memo`, `/wb-recall`: memory-manager를 통해 어댑터에 위임
- `/wb-start`, `/wb-run`: task-manager를 통해 어댑터에 위임
- 기존 사용 방식은 변경 없습니다
